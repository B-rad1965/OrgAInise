description: Naming conventions for Zod schemas and React Query hooks generated by Orval from the OpenAPI spec.
---

## Rule
Generated Zod names follow `{PascalCase(operationId)}Body` for request bodies and `{PascalCase(operationId)}Response` for responses.

**Why:** Orval derives names from the OpenAPI `operationId` field — changing the operationId changes the generated name.

**How to apply:**
- After any OpenAPI spec change: `pnpm --filter @workspace/api-spec run codegen`
- React Query hooks: `use{PascalCase(operationId)}` (e.g. operationId `getProjects` → `useGetProjects`)
- Zod schemas live in `lib/api-zod/src/`; hooks in `lib/api-client-react/src/`
- Do not hand-edit generated files — they are overwritten on next codegen run.
- The `info.title` in `openapi.yaml` controls the generated filenames — do not change it.
