## PHASE 2: PROPOSE SAFE DESIGN

After the audit, propose a safe implementation plan for a future feature called:

Memory Revision / Canon Update

The plan should be review-first and non-destructive.

Required behavior:

1. User opens a “Revise Memories” tool inside a project.

2. User describes the change in plain language.

Examples:

* “Keal is no longer canon. Find all memories related to Keal.”
* “Aurora’s deepest fear is abandonment, not intimacy. Update affected memories.”
* “The old magic system is obsolete. Archive memories about shard-based teleportation.”
* “Rename Keal to Caelen everywhere, but let me review first.”

3. App searches existing project memories and session history for related content.

4. AI analyzes results and proposes actions.

Possible proposed actions:

* Keep unchanged
* Archive old memory
* Delete memory
* Rewrite memory
* Rename entity
* Move category
* Change importance level

5. User sees a review screen before anything happens.

Each proposed change should show:

* Original memory
* Proposed action
* Proposed replacement text if rewriting
* Reason for suggestion
* Checkbox or approve/reject controls

6. User can approve individually or approve all.

7. Before applying approved changes, create a rollback snapshot.

8. Applying changes should never happen automatically.

9. Old canon should preferably be archived rather than permanently deleted by default.

10. User should be able to view archived memories and restore them later.

## AI SAFETY REQUIREMENTS

AI should never directly mutate data.

AI only produces suggestions.

The app applies changes only after user approval.

AI output must be structured JSON, not free-form text, so the review UI can safely display proposed changes.

Suggested output shape:

{
"summary": "Short explanation of what was found",
"matches": [
{
"memoryId": "abc123",
"currentText": "...",
"proposedAction": "archive | rewrite | keep | delete | rename | recategorize",
"proposedText": "...",
"reason": "...",
"confidence": "low | medium | high"
}
]
}

If confidence is low, default to “review carefully” and do not auto-select.
